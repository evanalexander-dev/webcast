Webcast — Privacy Policy

Last updated: May 2026

Webcast is an internal church live streaming control application. This privacy policy explains how the application handles data in connection with its use of the YouTube API.

What Webcast Does

Webcast is a self-hosted application used internally by a single church organization to automate the scheduling, starting, and stopping of YouTube Live broadcasts for Sunday meetings. It is not a public-facing service and has no end users other than the church's own administrators.

YouTube API Usage

Webcast uses the YouTube Data API v3 to:

  • Create and schedule YouTube Live broadcasts
  • Manage live stream ingestion keys
  • Transition broadcasts between states (scheduled, live, complete)
  • Retrieve view counts for completed broadcasts
  • Delete completed broadcast recordings after processing

Webcast's use of YouTube API data is limited to these functions and does not involve any collection, sharing, or sale of user data to third parties.

Data Collected and Stored

Webcast stores the following data locally on its own server:

  • YouTube OAuth refresh tokens for authorized channel accounts
  • YouTube broadcast and stream IDs for scheduled events
  • View count statistics fetched from completed broadcasts before deletion

No data is transmitted to any third party other than Google's YouTube API as required for the application to function.

Data Retention

OAuth tokens are stored only as long as the associated YouTube channel remains connected to the application. Broadcast statistics are retained locally for reporting purposes. No personally identifiable information about viewers is collected or stored.

Google API Services User Data Policy

Webcast's use of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

Contact

This application is maintained internally. For questions about this privacy policy, contact the application administrator.
